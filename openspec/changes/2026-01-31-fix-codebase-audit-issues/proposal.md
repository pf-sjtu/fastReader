# 修复代码库审计问题

## 变更概述

基于代码库审计报告，系统性修复功能、流程、UI 三个维度的关键问题。

## 背景与动机

经过全面审计，发现以下关键问题需要修复：

### 功能问题
1. **package.json name 不正确** - 使用的是模板名称而非项目实际名称
2. **缓存策略不合理** - 999天过期时间几乎等于永久，可能导致存储溢出
3. **国际化覆盖不全** - 多处硬编码中文，缺乏英文翻译
4. **AI 配置系统混乱** - 两套并行配置接口，维护困难

### 流程问题
1. **App.tsx 过于庞大** - 1638行，包含30+个state，难以维护
2. **文件处理流程繁琐** - 需要多次点击才能开始处理
3. **缺少流程引导** - 新用户不知道操作顺序

### UI 问题
1. **响应式设计缺陷** - 固定宽度布局，移动端无法使用
2. **组件样式不一致** - gap、按钮变体不统一
3. **加载状态体验差** - 单进度条，无法取消

## 目标

1. 修复高优先级功能缺陷
2. 改善核心用户流程
3. 提升 UI 一致性和响应式体验

## 影响范围

| 文件 | 变更类型 | 影响 |
|------|----------|------|
| package.json | MODIFY | 包名更正 |
| src/services/cacheService.ts | MODIFY | 缓存策略调整 |
| src/i18n/locales/en.json | MODIFY | 补充翻译 |
| src/components/*.tsx | MODIFY | 提取硬编码文本 |
| src/App.tsx | MODIFY | 代码结构优化（暂不拆分） |

## 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| 缓存过期导致用户数据丢失 | 低 | 中 | 保留现有缓存，新数据使用新策略 |
| i18n 键缺失导致显示问题 | 中 | 低 | 添加 fallback 处理 |
| 布局调整破坏现有样式 | 低 | 中 | 渐进式调整，保持向后兼容 |

## 验收标准

- [ ] package.json name 改为 "fastreader"
- [ ] 缓存过期时间改为 7 天
- [ ] 所有硬编码中文提取到 i18n
- [ ] 英文翻译文件完整
- [ ] 布局在 1024px 以上屏幕正常显示
